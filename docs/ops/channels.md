# 渠道管理

渠道是连接上游 API 提供商的桥梁。

---

## 渠道类型

| 类型 | 说明 | 示例 |
|:-----|:-----|:-----|
| OpenAI 兼容 | OpenAI API 兼容格式 | OpenAI、Azure |
| Anthropic | Anthropic Claude API | Claude |
| 自定义 | 自定义 API 格式 | 其他三方 API |

---

## 添加渠道

1. 进入 **运营管理 → 渠道管理**
2. 点击 **添加渠道**
3. 填写配置：

| 配置项 | 说明 | 必填 |
|:------|:-----|:----:|
| 名称 | 渠道显示名称 | ✓ |
| 类型 | 渠道类型 | ✓ |
| Base URL | 上游 API 地址 | ✓ |
| API Key | 上游 API 密钥 | ✓ |
| 模型映射 | 模型名称映射 | - |

### 示例配置

**OpenAI 兼容渠道：**

```
名称: OpenAI API
类型: OpenAI 兼容
Base URL: https://api.openai.com/v1
API Key: sk-xxxxxxxxxxxx
```

---

## 模型映射

如果上游模型名称与系统模型不一致，需要配置映射：

```json
{
  "gpt-4": "glm-4",
  "gpt-3.5-turbo": "glm-3-turbo"
}
```

> [!NOTE]
> 映射后，上游的 `gpt-4` 会绑定到系统的 `glm-4` 模型。

---

## 优先级设置

数字越小优先级越高：

| 优先级 | 适用场景 |
|:------:|:---------|
| 1 | 主用渠道，最高优先级 |
| 2 | 备用渠道 |
| n | 按需配置 |

---

## 状态管理

| 状态 | 说明 |
|:-----|:-----|
| 启用 | 正常可用，接收请求 |
| 禁用 | 暂停使用，不接收请求 |
| 自动 | 根据健康状态自动切换 |

---

## 自动管理

开启 **自动管理** 后，系统自动监控：

| 监控指标 | 阈值 | 处理 |
|:---------|:-----|:-----|
| 请求成功率 | < 90% | 自动禁用 |
| 响应时间 | > 30s | 记录警告 |
| 错误率 | > 10% | 自动禁用 |

> [!TIP]
> 自动禁用的渠道会在一定时间后自动尝试恢复。
