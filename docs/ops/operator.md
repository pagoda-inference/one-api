## 用户管理

管理系统用户，编辑用户信息和权限。

### 用户列表

| 字段 | 说明 |
|:-----|:-----|
| 用户名 | 显示名称或登录名 |
| 邮箱 | 用户邮箱 |
| Group | 用户分组，影响计费倍率 |
| 角色 | 普通用户/管理员/超级管理员 |
| 状态 | 正常/禁用/已删除 |
| 额度 | 当前剩余额度 |
| 已用额度 | 累计消耗额度 |
| 请求次数 | API 调用次数 |

### 角色说明

| 角色值 | 名称 | 权限 |
|:------:|:-----|:-----|
| 1 | 普通用户 | 使用 API |
| 10 | 管理员 | 运营管理 |
| 100 | 超级管理员 | 全部权限 |

### 编辑用户

点击操作栏「编辑」可修改：

- **分组（Group）**：如 `bedi`, `default`
- **额度**：-1 = 无限制，0 = 无额度
- **角色**：普通用户/管理员/超级管理员
- **状态**：正常/禁用

### 批量操作

支持批量启用、禁用、删除用户：

1. 勾选目标用户
2. 选择批量操作
3. 确认执行

> [!DANGER]
> 批量删除不可恢复，请谨慎操作。

### 用户分组

通过 Group 对用户进行分组，不同分组可设置不同计费倍率。

示例分组：
- `default` - 默认分组，倍率 1.0
- `vip` - VIP 用户，倍率 0.8
- `enterprise` - 企业用户，倍率 0.7

> [!NOTE]
> 分组倍率配置在 **系统设置 → 分组倍率** 中。

## 系统设置

### 计费公式

```
消耗额度 = (输入 Token × 输入价格 / 1000 + 输出 Token × 输出价格 / 1000) × 分组倍率
```

#### 变量说明

| 变量 | 来源 | 说明 |
|:-----|:-----|:-----|
| 输入 Token | API 请求 | 用户发送的 Token 数 |
| 输出 Token | API 响应 | 模型生成的 Token 数 |
| 输入价格 | 模型配置 | 元 / 千 Token |
| 输出价格 | 模型配置 | 元 / 千 Token |
| 分组倍率 | 用户 Group | 由 Group 配置决定 |

#### 计算示例

**场景：**

| 项目 | 值 |
|:----|:---|
| 用户 Group | vip（倍率 0.8） |
| 模型 | glm-4 |
| 输入价格 | 0.1 元/K |
| 输出价格 | 0.2 元/K |
| 输入 Token | 1000 |
| 输出 Token | 500 |

**计算：**

```
消耗额度 = (1000 × 0.1 / 1000 + 500 × 0.2 / 1000) × 0.8
         = (0.1 + 0.1) × 0.8
         = 0.16 元
```

### 分组倍率

配置不同用户分组的价格倍率。

```json
{
  "default": 1.0,
  "vip": 0.8,
  "enterprise": 0.7
}
```

JSON 格式，键为分组名称，值为倍率。

#### 分组说明

| 分组 | 倍率 | 说明 |
|:----:|:----:|:-----|
| default | 1.0 | 默认分组，不打折 |
| vip | 0.8 | VIP 用户，8 折 |
| enterprise | 0.7 | 企业用户，7 折 |

> [!NOTE]
> 告警邮箱和 Webhook 至少配置一项。

#### 告警类型

| 类型 | 触发条件 | 严重程度 |
|:-----|:---------|:---------|
| 渠道故障 | 渠道请求失败率过高 | 高 |
| 系统异常 | 系统运行出现错误 | 高 |
| 额度预警 | 用户额度低于阈值 | 中 |

#### Webhook 通知

告警信息通过 HTTP POST 发送到配置的 Webhook URL：

```json
{
  "type": "alert",
  "level": "warning",
  "title": "渠道故障告警",
  "message": "渠道 OpenAI-API 请求失败率超过 90%",
  "channel_id": 1,
  "channel_name": "OpenAI-API",
  "time": "2024-01-01 12:00:00"
}
```

Webhook 请求包含 `X-Signature` 头，可用于验证请求真实性。

#### 告警阈值

| 指标 | 默认阈值 | 说明 |
|:-----|:--------:|:-----|
| 请求失败率 | > 10% | 触发渠道故障告警 |
| 响应时间 | > 30s | 记录延迟警告 |
| 渠道禁用 | - | 自动禁用高故障渠道 |

### 性能配置

| 配置项 | 说明 | 默认值 |
|:-------|:-----|:------:|
| 最大并发请求数 | 同时处理的最大请求数 | 100 |
| 请求队列长度 | 请求等待队列长度 | 200 |
| 等待超时 | 请求在队列中等待的最长时间 | 120 秒 |
| 健康检查间隔 | 渠道健康检查间隔 | 60 秒 |
| 熔断失败阈值 | 触发熔断的连续失败次数 | 10 次 |
| 熔断超时 | 熔断恢复时间 | 60 秒 |
| 请求超时 | 单次请求超时时间 | 120 秒 |

#### 熔断机制

当渠道连续失败次数达到阈值时，该渠道会被暂时禁用：

```
失败 → 失败 → 失败 → ... → 达到阈值 → 熔断 → 等待恢复时间 → 尝试恢复
```

**恢复逻辑：**
- 熔断超时后，系统自动尝试恢复
- 如果恢复后仍然失败，重新熔断
- 连续恢复失败次数过多会延长熔断时间

#### 队列机制

当并发请求数达到上限时，新请求会进入队列等待：

| 情况 | 处理方式 |
|:-----|:---------|
| 队列未满 | 进入队列等待 |
| 队列已满 | 返回 503 错误 |
| 等待超时 | 请求被拒绝 |

#### 调优建议

| 场景 | 推荐配置 |
|:-----|:---------|
| 小规模使用 | 默认配置即可 |
| 大规模并发 | 增大并发数和队列长度 |
| 敏感业务 | 减小超时时间，快速失败 |
| 高可用要求 | 启用自动管理和熔断 |

> [!WARNING]
> 过大或过小的配置都可能影响系统稳定性，请根据实际负载调整。

## 认证配置

支持多种登录方式，包括密码登录和第三方 OAuth。

### 支持的 OAuth 类型

| 类型 | 说明 |
|:-----|:-----|
| 飞书 OAuth | 企业飞书账号登录 |
| GitHub OAuth | GitHub 账号登录 |
| OIDC | 标准 OIDC 单点登录 |

> [!NOTE]
> 默认全部关闭，需要配置并开启才能使用。

### 环境变量配置

| 配置项 | 环境变量 | 说明 | 默认值 |
|:------|:--------|:-----|:------:|
| 密码登录 | `PASSWORD_LOGIN_ENABLED` | 开启密码登录 | true |
| 注册功能 | `REGISTER_ENABLED` | 允许新用户注册 | true |
| GitHub OAuth | `GITHUB_OAUTH_ENABLED` | GitHub 登录 | false |
| 飞书 OAuth | `LARK_OAUTH_ENABLED` | 飞书登录 | false |
| OIDC 登录 | `OIDC_ENABLED` | OIDC 单点登录 | false |

### 飞书 OAuth

#### 配置步骤

1. 在[飞书开放平台](https://open.feishu.cn/)创建应用
2. 获取 App ID 和 App Secret
3. 配置回调地址：

```
https://your-domain.com/oauth/lark/callback
```

#### 环境变量

```bash
LarkClientId=your_app_id
LarkClientSecret=your_app_secret
LarkOAuthEnabled=true
```

#### 飞书应用权限

需要添加以下权限：
- `contact:user.employee_id:readonly`
- `contact:user.base:readonly`

### GitHub OAuth

#### 配置步骤

1. 在 [GitHub Developer Settings](https://github.com/settings/developers) 创建 OAuth App
2. 获取 Client ID 和 Client Secret
3. 配置回调地址：

```
https://your-domain.com/oauth/github/callback
```

#### 环境变量

```bash
GitHubClientId=your_client_id
GitHubClientSecret=your_client_secret
GitHubOAuthEnabled=true
```

### OIDC 配置

#### 配置项

| 环境变量 | 说明 |
|:---------|:-----|
| OidcIssuer | OIDC Issuer URL |
| OidcClientId | Client ID |
| OidcClientSecret | Client Secret |
| OidcRedirectUrl | 回调地址 |

#### 必要的 Scope

```json
["openid", "profile", "email"]
```

#### 配置示例

```bash
OIDC_ENABLED=true
OIDC_ISSUER=https://your-idp.com
OIDC_CLIENT_ID=your_client_id
OIDC_CLIENT_SECRET=your_client_secret
OIDC_REDIRECT_URL=https://your-domain.com/oauth/oidc/callback
```

### 回调地址

在 OAuth 应用配置中设置的回调地址必须与系统配置一致：

| OAuth 类型 | 回调地址 |
|:-----------|:---------|
| 飞书 | `/oauth/lark/callback` |
| GitHub | `/oauth/github/callback` |
| OIDC | `/oauth/oidc/callback` |

> [!WARNING]
> 回调地址不匹配会导致 OAuth 登录失败。

```bash
GitHubClientId=your_client_id
GitHubClientSecret=your_client_secret
```

### 飞书 OAuth 所需环境变量

```bash
LarkClientId=your_client_id
LarkClientSecret=your_client_secret
```

## 支付配置

### 支付方式

| 方式 | 说明 | 状态 |
|:-----|:-----|:-----|
| 支付宝 | 扫码支付 | 默认关闭 |
| 微信支付 | 扫码支付 | 默认关闭 |

### 支付宝配置

#### 申请流程

1. 签约支付宝商家服务
2. 获取以下凭证：
   - App ID（应用编号）
   - 应用私钥
   - 支付宝公钥

#### 环境变量

```bash
ALIPAY_ENABLED=true
# 支付宝相关凭证
```

### 微信支付配置

#### 申请流程

1. 签约微信支付商户服务
2. 获取以下凭证：
   - 商户号
   - API 密钥 或 API 证书

#### 环境变量

```bash
WECHAT_PAY_ENABLED=true
# 微信支付相关凭证
```

### 汇率配置

| 配置项 | 说明 | 默认值 |
|:------|:-----|:------:|
| PaymentExchangeRate | 支付汇率 | 1.0 |

> [!NOTE]
> 用于外币支付时转换为人民币结算。

### 支付流程

```
用户选择金额 → 创建订单 → 扫码支付 → 支付回调 → 额度到账
```

| 步骤 | 说明 |
|:-----|:-----|
| 1. 创建订单 | 生成支付订单和二维码 |
| 2. 扫码支付 | 用户使用支付宝/微信扫码 |
| 3. 支付回调 | 支付成功后接收回调通知 |
| 4. 额度到账 | 系统自动增加用户额度 |

### 常见问题

**Q: 支付回调失败怎么办？**

A: 系统会自动重试，如持续失败请联系技术支持。

**Q: 可以退款吗？**

A: 可以，请联系客服处理退款事宜。
