# 用量明细

查看 API 使用情况和消耗统计。

---

## 用量汇总

在 **用量明细** 页面顶部查看汇总数据：

| 指标 | 说明 |
|:-----|:-----|
| 总请求数 | 累计 API 调用次数 |
| 总消耗额度 | 累计消耗的额度（Quota） |
| 总 Token 数 | 累计使用 Token 总数（输入+输出） |

> [!TIP]
> 额度不足时及时充值，避免影响业务。

---

## 用量趋势

查看近期的用量趋势图表：

- **按日统计** - 每日用量变化
- **按模型统计** - 各模型用量占比

---

## 按模型统计

点击 **按模型** 标签查看各模型使用情况：

| 字段 | 说明 |
|:-----|:-----|
| 模型名称 | 使用的模型 |
| 请求次数 | 调用次数 |
| 消耗额度 | 消耗的 Quota |
| 输入 Token | 累计输入 Token |
| 输出 Token | 累计输出 Token |

## 用量明细列表

点击 **用量明细** 标签查看详细调用记录：

| 字段 | 说明 |
|:-----|:-----|
| 时间 | 请求时间 |
| 模型 | 使用的模型 |
| 类型 | 请求类型（chat/embedding 等） |
| 输入 Token | 消耗的输入 Token |
| 输出 Token | 消耗的输出 Token |
| 消耗额度 | 实际消耗的 Quota |

> [!NOTE]
> 明细记录保留最近 30 天的数据。

## 常见问题

**Q: 为什么显示的用量和实际不符？**

A: 可能原因：
- 预扣额度与实际消耗有差异（已调整）
- 部分请求失败未计入
- 并非所有模型都启用用量记录

**Q: 如何降低用量？**

A: 建议：
- 适当降低 `max_tokens` 限制
- 减少不必要的上下文对话
- 使用流式输出（streaming）
