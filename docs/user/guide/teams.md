# 团队管理

团队管理用于创建和管理团队，方便对成员进行统一配额分配和权限控制。

---

## 团队层级

支持多级层级结构：

```
公司
└── 部门
    └── 团队
```

> [!NOTE]
> 具体层级结构由管理员配置。

---

## 角色权限

| 角色 | 权限等级 | 可执行操作 |
|:-----|:--------:|:-----------|
| 管理员 | - | 管理团队设置、成员、额度分配 |
| 成员 | - | 使用 API、查看用量 |
| 观察者 | - | 仅查看数据，无使用权限 |

---

## 邀请成员

1. 进入 **运营管理 → 团队管理**
2. 选择目标团队
3. 点击 **邀请成员**
4. 填写信息：
   - 成员邮箱或用户 ID
   - 角色权限
5. 发送邀请

> [!WARNING]
> 被邀请人需已是系统用户。

---

## 额度分配

### 团队配额

管理员可以为团队设置总配额，团队成员共享使用。

### 分配方式

| 方式 | 说明 |
|:-----|:-----|
| 按成员分配 | 每个成员分配固定额度，互不影响 |
| 共享额度池 | 团队成员共享团队总配额 |

---

## 使用限制

可以设置团队级别的使用限制：

| 限制项 | 说明 |
|:-------|:-----|
| 总请求频率 | 团队所有成员合计的 RPM 上限 |
| 总 Token 限制 | 团队所有成员合计的 TPM 上限 |
| 允许模型 | 团队成员可使用的模型列表 |

---

## 审计日志

管理员可以查看团队内的操作记录：

| 记录类型 | 说明 |
|:---------|:-----|
| 成员操作日志 | 成员登录、使用记录 |
| 额度变动记录 | 配额增加/消耗明细 |
| API 调用记录 | 详细调用日志 |
